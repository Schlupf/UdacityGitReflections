	Fehler, der vorher nicht da war schneller finden.

How could having easy access to the entire history of a file make you a better
programmer in the long term?

  	Dann kann ich mit Funktionen experimentieren und sie nach Belieben
	durch alte Versionen ersetzen. Durch Vergleichen der verschiedenen 
	Versionen kann man das Beste herausfinden. 

What do you think are the pros and cons of manually choosing when to create a
commit, like you do in Git, vs having versions automatically saved, like Google
docs does?
    
	Pro: Man bekommt dur manuelles speichern nur sinnvolle Speicherpunkte
	an denen logische Blöcke vo0n Änderungen abgeschlossen sind.

	Con: Man kann vergessen zu Speichern und durch einen Absturz Daten
	verlieren.

Why do you think some version control systems, like Git, allow saving multiple
files in one commit, while others, like Google Docs, treat each file separately?

	Weil bei git mehrere Dateien eine Funktionale einheit bilden, was bei
	Google Docs oder Dropbox anders ist.

How can you use the commands git log and git diff to view the history of files?
    
	Git log zeigt die Zusammenfassung der Änderungen mit der
	Commit-Message und dem Autor an. Mit diff lassen sich die Änderungen
	innerhalb der Datei herausfinden.
